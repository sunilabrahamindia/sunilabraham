<div class="timeline-section">

<div class="milestone milestone-style-1">
  <div class="milestone-icon icon-foundation">üéØ</div>
  <div class="milestone-content">
    <p>The Sunil Abraham Project was founded on <strong>2 October 2025</strong>, marking the point at which earlier planning turned into a sustained public documentation effort with a clear archival intent.</p>
  </div>
</div>

<div class="milestone milestone-style-2">
  <div class="milestone-icon icon-rhythm">‚è∞</div>
  <div class="milestone-content">
    <p>From the outset, the project adopted a visible and disciplined release rhythm. Version updates were published <strong>every Sunday at 8:00 am IST</strong>, creating a steady public cadence that shaped how work was planned, reviewed, and recorded.</p>
  </div>
</div>

<div class="milestone milestone-style-3">
  <div class="milestone-icon icon-structure">üèóÔ∏è</div>
  <div class="milestone-content">
    <p>The early phase focused on laying down structural fundamentals. Attention was given to layouts, metadata, URLs, and category logic so that different forms of material could coexist without confusion as the site expanded.</p>
  </div>
</div>

<div class="milestone milestone-style-4">
  <div class="milestone-icon icon-growth">üìà</div>
  <div class="milestone-content">
    <p>Over the following weeks, the archive grew rapidly as publications, media articles, and references were located, verified, and prepared for publication. By the end of December 2025, the project had crossed <strong>365 articles</strong>, reflecting the scale of material that had previously been dispersed across platforms.</p>
  </div>
</div>

<div class="milestone milestone-style-1">
  <div class="milestone-icon icon-coherence">üß©</div>
  <div class="milestone-content">
    <p>As content accumulated, questions of coherence became central. Recurring subjects, long-running debates, and sustained engagements began to appear across multiple articles and media mentions.</p>
  </div>
</div>

<div class="milestone milestone-style-2">
  <div class="milestone-icon icon-clusters">üîó</div>
  <div class="milestone-content">
    <p>To address this, <strong>clusters</strong> were introduced as a core organisational tool. Clusters bring together related material under a single navigable page, helping readers move across people, publications, and themes without flattening individual articles.</p>
  </div>
</div>

<div class="milestone milestone-style-3">
  <div class="milestone-icon icon-archive">üìö</div>
  <div class="milestone-content">
    <p>Each cluster functions as a small curated archive, offering background context and structured pathways through related content while preserving the independence of each page.</p>
  </div>
</div>

<div class="milestone milestone-style-4">
  <div class="milestone-icon icon-organization">üé®</div>
  <div class="milestone-content">
    <p>The move toward clusters marked a shift from accumulation to organisation, ensuring that growth did not come at the cost of readability or long-term usability.</p>
  </div>
</div>

<div class="milestone milestone-style-1">
  <div class="milestone-icon icon-portal">üèõÔ∏è</div>
  <div class="milestone-content">
    <p>Dedicated portals further deepened this approach. The <strong>A. M. A. Ayrookuzhiel Portal</strong> became an important milestone, bringing together biographical material, writings, archival references, and interpretive studies related to Athanasius Mathen Abraham Ayrookuzhiel.</p>
  </div>
</div>

<div class="milestone milestone-style-2">
  <div class="milestone-icon icon-research">üî¨</div>
  <div class="milestone-content">
    <p>Designed as a research-oriented space, the portal allows students, researchers, and family members to explore his life and work without navigating across scattered categories.</p>
  </div>
</div>

<div class="milestone milestone-style-3">
  <div class="milestone-icon icon-ai">ü§ñ</div>
  <div class="milestone-content">
    <p>Toward the end of the Version 1.0 cycle, work began on the <strong>Artificial Intelligence Portal</strong>. This portal consolidates Sunil Abraham's engagement with artificial intelligence as a policy domain, regulatory challenge, and subject of public debate.</p>
  </div>
</div>

<div class="milestone milestone-style-4">
  <div class="milestone-icon icon-future">üöÄ</div>
  <div class="milestone-content">
    <p>Together, the weekly release rhythm, clusters, and portals reflect a maturing editorial approach. They allow the project to remain transparent, navigable, and sustainable as a long-term public archive.</p>
  </div>
</div>

<script>
// Scroll animation observer
const observerOptions = {
  threshold: 0.1,
  rootMargin: '0px 0px -50px 0px'
};

const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.style.opacity = '1';
      entry.target.style.transform = 'translateX(0)';
    }
  });
}, observerOptions);

document.querySelectorAll('.milestone').forEach(milestone => {
  observer.observe(milestone);
});

// Stat counter animation
const animateCounter = (element, target) => {
  let current = 0;
  const increment = target / 50;
  const timer = setInterval(() => {
    current += increment;
    if (current >= target) {
      element.textContent = target + (target >= 365 ? '+' : '');
      clearInterval(timer);
    } else {
      element.textContent = Math.floor(current) + (target >= 365 ? '+' : '');
    }
  }, 30);
};

const statObserver = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting && !entry.target.classList.contains('animated')) {
      const number = entry.target.querySelector('.stat-number');
      const target = parseInt(number.textContent);
      animateCounter(number, target);
      entry.target.classList.add('animated');
    }
  });
}, { threshold: 0.5 });

document.querySelectorAll('.stat-card').forEach(card => {
  statObserver.observe(card);
});
