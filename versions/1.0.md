**The Sunil Abraham Project** (TSAP) started in October 2025 as an effort to bring together writing, research, and public engagement that had long existed across different platforms and institutions. The aim was not to repackage this work, but to document it carefully â€” preserving context, authorship, and continuity while making it easier to read and navigate in one place.

Over the following two months, the project followed a steady weekly rhythm that helped structure both content creation and organisation. As patterns emerged across articles and media mentions, clusters and dedicated portals were introduced. On 1 January 2026, TSAP reached **Version 1.0**, marking a transition from rapid building to long-term stewardship and refinement.

<div class="milestone milestone-style-1">
  <div class="milestone-icon icon-foundation">ğŸ¯</div>
  <div class="milestone-content">
    <p><strong>The Sunil Abraham Project</strong> (TSAP) was founded on 2 October 2025, marking the point at which earlier planning turned into a sustained public documentation effort with a clear archival intent.</p>
  </div>
</div>

<div class="milestone milestone-style-2">
  <div class="milestone-icon icon-rhythm">â°</div>
  <div class="milestone-content">
    <p>From the outset, the project adopted a visible and disciplined release rhythm. Version updates were <a href="/versions/">published</a> every Sunday at 8:00 am IST, creating a steady public cadence that shaped how work was planned, reviewed, and recorded.</p>
  </div>
</div>

<div class="milestone milestone-style-3">
  <div class="milestone-icon icon-structure">ğŸ—ï¸</div>
  <div class="milestone-content">
<p>The early phase focused on laying down structural fundamentals. Built on <a href="https://github.com/sunilabrahamindia/sunilabraham/" rel="noopener noreferrer nofollow">GitHub Pages</a> using <a href="https://jekyllrb.com/" rel="noopener noreferrer nofollow">Jekyll</a>, attention was given to layouts, metadata, URLs, and <a href="/categories/">category logic</a>, ensuring that different forms of material could coexist cleanly and remain navigable as the site expanded.
</p>

  </div>
</div>

<div class="milestone milestone-style-4">
  <div class="milestone-icon icon-growth">ğŸ“ˆ</div>
  <div class="milestone-content">
    <p>Over the following weeks, the archive grew rapidly as publications, media articles, and references were located, verified, and prepared for publication. By the end of December 2025, the project had crossed <strong>365 articles</strong>, reflecting the scale of material that had previously been dispersed across platforms.</p>
  </div>
</div>

<div class="milestone milestone-style-1">
  <div class="milestone-icon icon-coherence">ğŸ§©</div>
  <div class="milestone-content">
    <p>As content accumulated, questions of coherence became central. Recurring subjects, long-running debates, and sustained engagements began to appear across multiple articles and media mentions.</p>
  </div>
</div>

<div class="milestone milestone-style-2">
  <div class="milestone-icon icon-clusters">ğŸ”—</div>
  <div class="milestone-content">
    <p>To address this, <a href="/clusters/">clusters</a> were introduced as a core organisational tool. Clusters bring together related material under a single navigable page, helping readers move across people, publications, and themes without flattening individual articles.</p>
  </div>
</div>

<div class="milestone milestone-style-3">
  <div class="milestone-icon icon-archive">ğŸ“š</div>
  <div class="milestone-content">
    <p>Each cluster functions as a small curated archive, offering background context and structured pathways through related content while preserving the independence of each page.</p>
  </div>
</div>

<div class="milestone milestone-style-4">
  <div class="milestone-icon icon-accessibility">ğŸ§­</div>
  <div class="milestone-content">
    <p>
      From the outset, the project has been shaped by a mobile-first and accessibility-friendly approach. Design and content decisions prioritise readability on mobile screens, clear navigation, and inclusive presentation, ensuring the site remains usable and welcoming across devices, contexts, and abilities.
    </p>
  </div>
</div>

<div class="milestone milestone-style-1">
  <div class="milestone-icon icon-portal">ğŸ›ï¸</div>
  <div class="milestone-content">
<p>Dedicated portals further deepened this approach. The <a href="/amaa/portal">A. M. A. Ayrookuzhiel Portal</a> became an important milestone, bringing together biographical material, writings, archival references, and interpretive studies related to Athanasius Mathen Abraham Ayrookuzhiel.</p>
  </div>
</div>

<div class="milestone milestone-style-2">
  <div class="milestone-icon icon-research">ğŸ”¬</div>
  <div class="milestone-content">
<p>Designed as a research-oriented space, the portal allows students, researchers, and family members to explore his life and work without navigating across scattered categories.</p>
  </div>
</div>

<div class="milestone milestone-style-3">
  <div class="milestone-icon icon-ai">ğŸ¤–</div>

<p>Toward the end of the Version 1.0 cycle, work began on the <a href="/ai/">Artificial Intelligence Portal</a>. This portal consolidates Sunil Abraham's engagement with artificial intelligence as a policy domain, regulatory challenge, and subject of public debate.</p>
  </div>
</div>

<div class="milestone milestone-style-4">

const observerOptions = {
  threshold: 0.1,
  rootMargin: '0px 0px -50px 0px'
};

const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.style.opacity = '1';
      entry.target.style.transform = 'translateX(0)';
    }
  });
}, observerOptions);

document.querySelectorAll('.milestone').forEach(milestone => {
  observer.observe(milestone);
});

const animateCounter = (element, target) => {
  let current = 0;
  const increment = target / 50;
  const timer = setInterval(() => {
    current += increment;
    if (current >= target) {
      element.textContent = target + (target >= 365 ? '+' : '');
      clearInterval(timer);
    } else {
      element.textContent = Math.floor(current) + (target >= 365 ? '+' : '');
    }
  }, 30);
};

const startCountingCycle = (card) => {
  const number = card.querySelector('.stat-number');
  const target = parseInt(number.getAttribute('data-target') || number.textContent);
  number.setAttribute('data-target', target);
  
  number.textContent = '0';
  animateCounter(number, target);
  
  setTimeout(() => startCountingCycle(card), 7000);
};

const statObserver = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting && !entry.target.classList.contains('counting')) {
      entry.target.classList.add('counting');
      startCountingCycle(entry.target);
    }
  });
}, { threshold: 0.5 });

document.querySelectorAll('.stat-card').forEach(card => {
  statObserver.observe(card);
});
